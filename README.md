# AIsight Lunch Roulette

수요일 팀 점심용 랜덤 팀 배정 페이지입니다.

기존 점심 페이지에서 **룰렛 기능만 분리**한 전용 프로젝트이며,
운영 방식은 아래와 같습니다.

- 각 팀원이 **직접 출석 체크**
- 관리자는 **출석한 인원 기준으로만 룰렛 실행**
- 결과는 실시간으로 공유
- 초기화 / 재추첨은 관리자만 가능

---

## 목적

이 프로젝트는 그중 **수요일 룰렛 기능만 독립적으로 분리**하여,
별도 URL로 공유하고 유지보수하기 쉽게 만드는 것이 목표 

---

## 주요 기능

### 1. 개인 출석 체크
- 각 사용자가 자신의 이름으로 직접 출석 체크
- 출석 상태는 Firebase Realtime Database 에 저장
- 새로고침 후에도 유지
- 다른 사용자의 출석 상태도 실시간 반영

### 2. 관리자 전용 룰렛 실행
- 관리자 로그인 후에만 팀 뽑기 가능
- 출석 체크된 인원만 대상으로 팀 구성
- 최소 2명 이상 출석 시 실행 가능

### 3. 실시간 결과 공유
- 룰렛 결과는 Firebase 에 저장
- 같은 페이지를 보고 있는 모든 인원이 동일한 결과를 확인 가능

### 4. 재추첨 / 초기화
- 관리자만 결과 초기화 가능
- 필요 시 다시 팀 구성 가능

---

## 기존 코드에서 분리한 범위

기존 통합 페이지에서 아래 기능만 가져와 별도 프로젝트로 구성합니다.

- 수요일 룰렛 페이지 (`#wed-page`)
- 관리자 로그인 모달
- 팀 배정 알고리즘 (`makeTeams`)
- 룰렛 실행 (`spinRoulette`)
- 룰렛 결과 렌더링 (`renderRouletteResult`)

기존 월요일 메뉴 선택 / 자유 메뉴 / 여성 점심 팝업 등은 제외합니다.

---

## 개선 목표

기존 구현은 관리자가 직접 출석 인원을 선택한 뒤 룰렛을 돌리는 구조였습니다.
새 프로젝트에서는 아래와 같이 변경합니다.

### Before
- 관리자가 출석 대상 직접 체크
- 출석 정보가 관리자 브라우저 메모리(Set)에만 존재
- 다른 사용자는 출석 상태를 직접 반영할 수 없음

### After
- 각자가 자신의 출석을 직접 체크
- 출석 정보는 Firebase에 저장
- 관리자는 저장된 출석자 목록으로만 룰렛 실행
